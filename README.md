# PGP Stream Decrypt

A production-grade Python service that watches a directory for incoming PGP-encrypted files and decrypts them concurrently using a configurable thread pool.

## Features

- **Pure-Python OpenPGP decryption** — implements RFC 4880 directly using the `cryptography` library; no GPG binary required, no `pgpy` or `python-gnupg` dependencies
- **Streaming I/O** — AES-256-CFB decryption is processed in 64 KB chunks; RAM usage is independent of file size
- **MDC integrity verification** — SHA-1 Modification Detection Code checked on every file
- **Concurrent processing** — configurable `ThreadPoolExecutor` pool; multiple files decrypted in parallel
- **File-lifecycle management** — originals moved to `processed/` on success or `error/` on failure (atomic move)
- **Daily rotating logs** — 30-day retention, configurable level
- **INI-based configuration** — all paths, pool size, log level, and key material set in `config.properties`
- **CLI overrides** — `--workers`, `--log-level`, `--config` arguments override config file at runtime

## Quick Start

```bash
# 1. Create and activate virtual environment
python -m venv venv
source venv/Scripts/activate      # Windows
# source venv/bin/activate         # Linux / macOS

# 2. Install dependencies
pip install -r requirements.txt

# 3. Generate test keys and encrypted sample files
python pgps/testing/generate_test_data.py

# 4. Start the service
python src/main.py
```

Drop any `.pgp` or `.gpg` file into `pgps/input/` and the service picks it up immediately.

## Configuration

Edit `config.properties` to match your environment:

```ini
[paths]
input_dir     = pgps/input
output_dir    = pgps/output
processed_dir = pgps/processed
error_dir     = pgps/error

[decryption]
# PEM (PKCS#8 / PKCS#1) or PGP ASCII-armored (.asc) private key
private_key_path = pgps/testing/private_key.pem
# Passphrase protecting the key — leave empty for unprotected keys
passphrase =

[pool]
workers = 4

[logging]
level   = INFO      # DEBUG | INFO | WARNING | ERROR
log_dir = log
```

All paths are relative to the working directory where `main.py` is launched.

## CLI Usage

```bash
# Default — reads config.properties in the current directory
python src/main.py

# Override config file path
python src/main.py --config /etc/pgp-decrypt/prod.properties

# Override pool size and log level at runtime
python src/main.py --workers 8 --log-level DEBUG
```

Stop the service with `Ctrl-C`; in-flight decryptions complete before shutdown.

## Supported PGP Formats

| Feature | Support |
|---|---|
| Public-key algorithm | RSA (PKCS#1 v1.5, packet tag 1) |
| Symmetric cipher | AES-128, AES-192, AES-256 |
| Integrity protection | SEIPD v1 (packet tag 18) with MDC SHA-1 |
| Compression | None, ZLIB (RFC 1950), ZIP (RFC 1951), BZIP2 |
| Private key format | PEM (PKCS#8 / PKCS#1) or PGP ASCII-armored |
| File extensions watched | `.pgp`, `.gpg`, `.asc` |

## Performance Benchmark

Measured on Windows 11, Python 3.9.13, single-threaded, RSA-2048 + AES-256-CFB, 1.5 GB file:

| Operation | Throughput | Wall time |
|---|---|---|
| Encrypt (stream write) | 343 MB/s | 4.47 s |
| Decrypt (stream read) | 197 MB/s | 7.79 s |
| SHA-256 verify | — | 2.8 s |

Decrypt throughput is lower than encrypt because decryption writes to a temporary file during inner-packet parsing before writing the final output (two sequential disk passes for the same data). Encryption is a single-pass stream directly to the output file.

To reproduce:

```bash
python pgps/testing/perf_test.py
```

The script generates the 1.5 GB plaintext on first run (cached on subsequent runs), encrypts it, decrypts it, verifies SHA-256 correctness, and prints the summary.

## Architecture

```
[FileWatcher]  ──submit──>  [WorkerPool]  ──decrypt──>  [PGPDecryptor]
 watchdog                    ThreadPool                   RFC 4880
 pgps/input/                 N workers                    RSA + AES-256-CFB
                                |                          MDC verify
                         success: processed/
                         failure: error/
```

**FileWatcher** (`src/watcher.py`): Uses `watchdog` to detect file-created events in `pgps/input/`. Also scans for pre-existing files at startup so files dropped before launch are not missed.

**WorkerPool** (`src/worker_pool.py`): Wraps `concurrent.futures.ThreadPoolExecutor`. Each submitted file runs `PGPDecryptor.decrypt_file` in a worker thread. On success the original is moved to `processed/`; on any exception it is moved to `error/`.

**PGPDecryptor** (`src/decryptor.py`): Parses PGP packets per RFC 4880. Finds the PKESK packet, decrypts the AES session key with the RSA private key (PKCS#1 v1.5), then stream-decrypts the SEIPD packet using OpenPGP CFB mode. Verifies the SHA-1 MDC before writing output. Handles Compressed Data (ZLIB / ZIP / BZIP2) and Literal Data inner packets.

**OpenPGP CFB implementation**: Standard AES-CFB cannot be used directly because RFC 4880 §13.9 specifies a two-phase mode with a resync after the (block\_size + 2) prefix bytes. The decryptor handles this by:
1. Manually decrypting the 18-byte prefix to verify the "quick check"
2. Re-initialising AES-CFB with `IV = ciphertext[2:18]` for the remainder

## Development Workflow

```bash
# Generate test keys and files
python pgps/testing/generate_test_data.py

# Start the watcher (Ctrl-C to stop)
python src/main.py

# In another terminal: copy test files to trigger processing
cp pgps/testing/*.pgp pgps/input/

diff pgps/output/sample.txt pgps/testing/sample.txt
```
